Monoid Algebra Showcase (NonCommutative / Projector / Unipotent)

This page showcases the three MonoidAlgebra variants supported by NCTSSoS.jl. These algebras provide the building blocks for modeling noncommutative polynomial optimization problems that arise in quantum information and many-body physics.

Capabilities Overview

Each monoid algebra type provides automatic simplification of operator products according to its defining relations:

Algebra TypeSimplification RuleTypical Use Case
NonCommutativeAlgebraNone (free words)General NC polynomial optimization
ProjectorAlgebraP¬≤ = P (idempotent)Projective measurements, NPA hierarchy
UnipotentAlgebraU¬≤ = ùüô (involution)Pauli observables, Bell inequalities

All three algebras share a common feature: site-based commutation. Operators acting on different physical sites automatically commute during canonicalization, while the order of operators on the same site is preserved.

Key Public APIs

Variable creation is done through these functions:

  • create_noncommutative_variables: Create free noncommutative generators
  • create_projector_variables: Create idempotent (P¬≤ = P) generators
  • create_unipotent_variables: Create involutory (U¬≤ = ùüô) generators

Each function returns a (registry, variable_groups) tuple, where the registry manages symbol ‚Üî index mappings and the variable groups provide convenient access to the created monomials.

References and Applications

  • NonCommutativeAlgebra (free words): The standard modeling layer for noncommutative polynomial optimization and SDP hierarchies. See Burgdorf‚ÄìKlep‚ÄìPovh (Burgdorf et al., 2016), Wang‚ÄìMagron (Wang and Magron, 2021), and the comprehensive treatment in (Magron and Wang, 2023).

  • ProjectorAlgebra (P¬≤ = P): Projection operators and projective measurements. In device-independent quantum information, projective measurements appear naturally in the Navascu√©s‚ÄìPironio‚ÄìAc√≠n (NPA) SDP hierarchy for bounding quantum correlations (Navascu√©s et al., 2007), (Navascu√©s et al., 2008). In many-body physics, projectors enforce local constraints and define subspaces; see Sachdev (Sachdev, 2011).

  • UnipotentAlgebra (U¬≤ = ùüô): Involutory (¬±1-valued) observables model self-adjoint unitaries like Pauli measurements. These are fundamental in quantum information (Nielsen and Chuang, 2010) and appear in Bell inequality optimization (Klep et al., 2024).

1) NonCommutativeAlgebra: Free Noncommutative Words

The NonCommutativeAlgebra represents free words with no algebraic relations other than site-based commutation. This is the most general monoid algebra type.

Creating Variables

Use create_noncommutative_variables with a vector of (prefix, subscripts) tuples. Each prefix group is assigned to a distinct site, enabling automatic commutation between groups.

reg_nc, (x, y) = create_noncommutative_variables([("x", 1:2), ("y", 1:2)])
(VariableRegistry with 4 variables: x‚ÇÅ, x‚ÇÇ, y‚ÇÅ, y‚ÇÇ, (NCTSSoS.NormalMonomial{NCTSSoS.NonCommutativeAlgebra, UInt8}[UInt8[0x05], UInt8[0x09]], NCTSSoS.NormalMonomial{NCTSSoS.NonCommutativeAlgebra, UInt8}[UInt8[0x0e], UInt8[0x12]]))

This creates:

  • Variables x[1], x[2] on site 1
  • Variables y[1], y[2] on site 2

A single generator is a NormalMonomial (immutable canonical form):

typeof(x[1])
NCTSSoS.NormalMonomial{NCTSSoS.NonCommutativeAlgebra, UInt8}

Site-Based Commutation

Operators on different sites commute automatically. The canonical form groups operators by site, preserving intra-site order:

lhs = x[2] * y[1] * x[1] * y[2]
rhs = x[2] * x[1] * y[1] * y[2]  # same canonical form

@assert lhs == rhs  # true: sites are sorted

repr_with_registry(reg_nc, mono1(lhs))  # "x‚ÇÇx‚ÇÅy‚ÇÅy‚ÇÇ"
"x‚ÇÇx‚ÇÅy‚ÇÅy‚ÇÇ"

Intra-Site Non-Commutativity

Within a single site, operators do NOT commute (no relations are imposed):

@assert x[1] * x[2] != x[2] * x[1]

Show both representations to see they differ:

(repr_with_registry(reg_nc, mono1(x[1] * x[2])),  # "x‚ÇÅx‚ÇÇ"
 repr_with_registry(reg_nc, mono1(x[2] * x[1])))  # "x‚ÇÇx‚ÇÅ"
("x‚ÇÅx‚ÇÇ", "x‚ÇÇx‚ÇÅ")

Inspecting the Canonical Word

Each index encodes both operator identity and site. You can decode them:

word = mono1(lhs).word

  • site: which physical site (1 or 2)

2) ProjectorAlgebra: Idempotent Operators (P¬≤ = P)

The ProjectorAlgebra models projection operators (idempotents). Any repeated application of the same projector collapses to a single instance.

Quantum Information Applications:

  • Projective measurements (PVM elements) in the NPA hierarchy (Navascu√©s et al., 2007), (Navascu√©s et al., 2008)
  • Local projectors like |0‚ü©‚ü®0| and |1‚ü©‚ü®1| for qubit measurements

Many-Body Physics Applications:

  • Local constraint projectors (e.g., Gutzwiller projectors)
  • Subspace projections in variational methods (Sachdev, 2011)

Creating Projector Variables

reg_proj, (P, Q) = create_projector_variables([("P", 1:2), ("Q", 1:2)])
(VariableRegistry with 4 variables: P‚ÇÅ, P‚ÇÇ, Q‚ÇÅ, Q‚ÇÇ, (NCTSSoS.NormalMonomial{NCTSSoS.ProjectorAlgebra, UInt8}[UInt8[0x05], UInt8[0x09]], NCTSSoS.NormalMonomial{NCTSSoS.ProjectorAlgebra, UInt8}[UInt8[0x0e], UInt8[0x12]]))
  • P[1], P[2] on site 1 (with P¬≤ = P)
  • Q[1], Q[2] on site 2 (with Q¬≤ = Q)

Idempotency Simplification

Repeated projectors automatically collapse:

@assert monomials(P[1] * P[1]) == [P[1]]        # P¬≤ = P
@assert monomials(P[1] * P[1] * P[1]) == [P[1]] # P¬≥ = P

Site Commutation with Idempotency

Cross-site operators commute, and idempotency is applied after canonicalization:

@assert monomials(Q[2] * P[1]) == monomials(P[1] * Q[2])  # site commutation

More complex example: commutation brings same-site operators together, then idempotency collapses repeated factors:

result = mono1(Q[1] * P[1] * Q[1] * P[1])
repr_with_registry(reg_proj, result)  # "P‚ÇÅQ‚ÇÅ" (both collapse)
"P‚ÇÅQ‚ÇÅ"

3) UnipotentAlgebra: Involutory Operators (U¬≤ = ùüô)

The UnipotentAlgebra models operators that square to the identity, such as Pauli matrices œÉ‚Çì, œÉ·µß, œÉ·µ§. This is the natural algebra for ¬±1-valued observables.

Quantum Information Applications:

  • Pauli observables in quantum computation (Nielsen and Chuang, 2010)
  • CHSH and other Bell inequalities with dichotomic measurements (Klep et al., 2024)
  • Stabilizer formalism and error correction

Many-Body Physics Applications:

  • Spin-¬Ω systems with Pauli operators
  • Ising model interactions

Creating Unipotent Variables

reg_unip, (U, V) = create_unipotent_variables([("U", 1:2), ("V", 1:2)])
const ID_UNIP = one(NormalMonomial{UnipotentAlgebra, eltype(indices(reg_unip))})
ùüô
  • U[1], U[2] on site 1 (with U¬≤ = ùüô)
  • V[1], V[2] on site 2 (with V¬≤ = ùüô)

Involution Simplification

Squared operators simplify to the identity:

@assert monomials(U[1] * U[1]) == [ID_UNIP]     # U¬≤ = ùüô
@assert monomials(U[1] * U[1] * U[1]) == [U[1]] # U¬≥ = U

Site Commutation with Involution

Cross-site commutation works as expected:

@assert monomials(V[2] * U[1]) == monomials(U[1] * V[2])

Cancellation After Canonicalization

Site-based canonicalization can bring paired operators together, enabling cancellation:

V‚ÇÅ U‚ÇÅ V‚ÇÅ U‚ÇÅ ‚Üí (after site sort) ‚Üí V‚ÇÅ V‚ÇÅ U‚ÇÅ U‚ÇÅ = ùüô

@assert monomials(V[1] * U[1] * V[1] * U[1]) == [ID_UNIP]

Simpler case: U‚ÇÅ¬≤ U‚ÇÇ = U‚ÇÇ

@assert monomials(U[1] * U[1] * U[2]) == monomials(U[2])

5) Practical Example: Building Polynomials

Monomial algebras integrate seamlessly with NCTSSoS.jl's polynomial optimization pipeline. Here's a simple example of building a polynomial objective:

Coefficients create polynomials from monomials

poly = 1.0 * x[1] * x[2] + 2.0 * y[1] - 3.0 * x[1] * y[1]
2.0 * UInt8[0x0e] + UInt8[0x05, 0x09] + -3.0 * UInt8[0x05, 0x0e]

Site commutation is automatic in polynomial arithmetic:

poly2 = 1.0 * y[1] * x[1] - 3.0 * x[1] * y[1]  # y‚ÇÅ and x‚ÇÅ commute!
monomials(poly2)  # simplified: terms combine if they have the same canonical form
1-element Vector{NCTSSoS.NormalMonomial{NCTSSoS.NonCommutativeAlgebra, UInt8}}:
 UInt8[0x05, 0x0e]
